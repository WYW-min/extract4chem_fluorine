1. split：只做切分，输出 sections.jsonl。每条必须有 doc_id / block_id / section_title / text / char_span or line_span。

2. route-*：只做任务路由，输入 sections.jsonl，输出 tasks_<route>.jsonl（如 tasks_char.jsonl、tasks_prop.jsonl、tasks_proc.jsonl）。

3. predict-*：只做模型调用，输入 tasks_*.jsonl，输出 pred_raw_*.jsonl（保留 raw response、prompt hash、model config、latency、token usage、error）。
- 主信号抽取
    > 输入：引言前所有内容 + 实验部分
    > 输出：文献元信息 + 实体主信号

- 属性抽取

    - 表征
        输入：身份标识 + 聚合物主锚点信息 + 定向段落文本

    - 性质

    - 工艺流程



4. parse-*：只做结构化解析与 schema 校验，输出 pred_parsed_*.jsonl（包含校验错误、字段级错误）。

5. merge：只做最终整合，输入 pred_parsed_*.jsonl，输出 final.json。



# 常见问题总结
1. 聚合物对象边界：是否明确要求把前驱体（PAA/PAAS）、油墨（PAAS/CNC ink）、最终气凝胶都作为同级对象输出？
是，判断逻辑:是原料还是合成出来的。

2. 同一体系不同配比（如 5.6/6.5/7.4 wt%）必须拆成独立对象吗？若正文只给“组级结论”，是否要复制到每个对象。
是

3. 聚合物分类编码是否有限制范围的词表（PI/CNC/PAA/PAAS...）？没有词表的话，不同标注员标注标准会不同。
无


4. 结构特征_L1/L2是否有固定枚举和定义？样例用了英文标签，需要确认是否强制该集合。
有三个，但不全面，可以自行总结

5. 性质.类别和性质.名称是否有标准字典（中英文映射）？如“Thermal Properties”与“热学性质”，是否允许混用。
无，允许混用

6. 测试条件是否要求“尽量全量保留原文字段”，还是要强制标准化字段集合？
根据文献情况，不好标准化的，尽量全量保留原文，写在"方法”

7. 工艺流程步骤切分标准：按“产物节点”切，还是按“实验段落动作”切；后处理是字符串还是数组。
按“产物节点”切;后处理可以是字符串，也可以是数组

8. 质量验收标准：是“字段覆盖率优先”还是“高精度宁缺毋滥优先”，以及可接受的漏召回/误召回阈值。
高精度宁缺毋滥优先，聚合物大类有的都分开，各聚合物里的详细信息接受少部分漏写。

9. TGA / DSC / DMA / TMA 这类热分析，后续统一归到 表征 还是 性质？
表征

10. CV / 电化学测试 / 气敏响应时间-恢复时间 这类结果，统一归到 表征 还是 性质/应用？
性质/应用
